# 🚀 快速参考 - DNS 记录添加指南

## 📖 记录字段说明

### 记录名 (Name) vs 记录值 (Value)

```
┌─────────────────────────────────────────────────┐
│ 完整域名: blog.example.com                      │
├─────────────────────────────────────────────────┤
│ 记录名 (Name):  blog                            │
│ 域名后缀:       example.com (from zone)        │
│ ────────────────────────────                    │
│ 记录值 (Value): 取决于记录类型                  │
└─────────────────────────────────────────────────┘
```

---

## 📋 各类型记录的字段填法

### 1️⃣ **A 记录** - IPv4 地址

| 字段 | 示例 | 说明 |
|------|------|------|
| 记录类型 | 1 (A) | 选择菜单 1 |
| 记录名 | www | 子域，不能有点号 |
| 记录值 | 192.0.2.1 | IPv4 地址 |
| TTL | 3600 | 默认值即可 |

**例子**：
```
域名：example.com
www 子域的 A 记录 192.0.2.1
→ 访问 www.example.com 时解析到 192.0.2.1
```

---

### 2️⃣ **AAAA 记录** - IPv6 地址

| 字段 | 示例 | 说明 |
|------|------|------|
| 记录类型 | 2 (AAAA) | 选择菜单 2 |
| 记录名 | www | 子域，不能有点号 |
| 记录值 | 2001:db8::1 | IPv6 地址 |
| TTL | 3600 | 默认值即可 |

**例子**：
```
IPv6 支持的 www 子域
→ 使用 IPv6 访问时解析到 2001:db8::1
```

---

### 3️⃣ **CNAME 记录** - 别名

| 字段 | 示例 | 说明 |
|------|------|------|
| 记录类型 | 3 (CNAME) | 选择菜单 3 |
| 记录名 | blog | 子域，不能有点号 |
| 记录值 | blog.wordpress.com | **完整域名** |
| TTL | 3600 | 默认值即可 |

**重点**：记录值是**完整域名**，可以有点号

**例子**：
```
将 blog.example.com 指向 WordPress 托管的 blog.wordpress.com
记录名: blog
记录值: blog.wordpress.com
```

---

### 4️⃣ **MX 记录** - 邮件服务

| 字段 | 示例 | 说明 |
|------|------|------|
| 记录类型 | 4 (MX) | 选择菜单 4 |
| 记录名 | @ | 顶级域，无子域 |
| 记录值 | mail.example.com | **完整域名** |
| TTL | 3600 | 默认值即可 |
| **优先级** | 10 | 1-65535，数字越小优先级越高 |

**重点**：多个 MX 记录时使用优先级

**例子**：
```
设置邮件服务器
主服务器: mail.example.com (优先级 10)
备用服务器: mail2.example.com (优先级 20)
```

---

### 5️⃣ **TXT 记录** - 文本记录

| 字段 | 示例 | 说明 |
|------|------|------|
| 记录类型 | 5 (TXT) | 选择菜单 5 |
| 记录名 | @ | 通常是顶级域 |
| 记录值 | v=spf1 include:example.com ~all | **任意文本** |
| TTL | 3600 | 默认值即可 |

**常见用途**：
- SPF: `v=spf1 include:sendgrid.net ~all`
- DKIM: `v=DKIM1; k=rsa; p=MIGfMA0BgQ...`
- DMARC: `v=DMARC1; p=none;`

---

### 6️⃣ **NS 记录** - 名字服务器

| 字段 | 示例 | 说明 |
|------|------|------|
| 记录类型 | 6 (NS) | 选择菜单 6 |
| 记录名 | subdomain | 子域的 NS 记录 |
| 记录值 | ns1.example.com | **完整域名** |
| TTL | 3600 | 默认值即可 |

**例子**：
```
将 subdomain.example.com 的 DNS 委托给其他 NS 服务器
记录名: subdomain
记录值: ns1.other-server.com
```

---

### 7️⃣ **SRV 记录** - 服务定位

| 字段 | 示例 | 说明 |
|------|------|------|
| 记录类型 | 7 (SRV) | 选择菜单 7 |
| 记录名 | _sip._tls | 服务标识符 |
| 记录值 | 0 5 5060 sipserver.example.com | **格式: 优先级 权重 端口 目标主机** |
| TTL | 3600 | 默认值即可 |

**例子**：
```
SIP 服务配置
_sip._tls.example.com → sipserver.example.com:5060
```

---

### 8️⃣ **CAA 记录** - 证书颁发授权

| 字段 | 示例 | 说明 |
|------|------|------|
| 记录类型 | 8 (CAA) | 选择菜单 8 |
| 记录名 | @ | 通常是顶级域 |
| 记录值 | 0 issue "letsencrypt.org" | **格式: 标志 标签 值** |
| TTL | 3600 | 默认值即可 |

**例子**：
```
只允许 Let's Encrypt 颁发证书
0 issue "letsencrypt.org"
```

## 🎯 快速添加步骤

### 步骤 1: 启动脚本
```bash
./bunnydns.sh
```

### 步骤 2: 选择操作
```
主菜单 → 选 5（单独添加记录）
```

### 步骤 3: 输入 Zone ID
```
请输入 Zone ID: 12345
```

### 步骤 4: 选择记录类型
```
1. A       - IPv4 地址
2. AAAA    - IPv6 地址
...
请输入选择 (1-8): 3  ← 选择你需要的类型
```

根据记录类型，脚本会给出相应的提示。按照提示填写即可。

### 步骤 6: 确认
```
正在添加 CNAME 记录...
✅ 操作成功 (HTTP 201)
```

## ❌ 常见错误与修正

| 错误 | 原因 | 修正 |
|------|------|------|
| `记录名只能包含字母、数字...` | 记录名有点号 | 移除点号，只保留子域 |
| `IPv4 地址不合法` | IP 地址超出范围或格式错 | 检查每个数字是否 0-255 |
| `无效的域名格式` | 域名格式错 | 确保是完整域名，如 example.com |
| `TTL 必须是数字` | TTL 包含非数字字符 | 输入纯数字，如 3600 |
