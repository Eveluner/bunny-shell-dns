# 🎬 交互流程演示 - 改进前后对比

## 📌 场景：添加 CNAME 记录（将 blog 子域指向 WordPress）

---

## ❌ **改进前的流程**

```bash
=== 添加 DNS 记录 ===
支持的记录类型: A, AAAA, CNAME, MX, TXT, NS, SRV, CAA

请输入记录类型: CNAME  # ⚠️ 需要手动输入，容易打错

请输入记录名 (@, www 或其他子域): blog.wordpress.com
❌ 记录名只能包含字母、数字、-、_、@ 或 *
# 用户疑惑，不知道哪里错了

# 用户重试
请输入记录名 (@, www 或其他子域): blog.wordpress.com
❌ 记录名只能包含字母、数字、-、_、@ 或 *

# 用户查看文档后...
请输入记录名 (@, www 或其他子域): blog
# ✅ 通过

请输入记录值: wordpress.com
请输入 TTL (默认 3600): 3600
正在添加记录...
✅ 操作成功 (HTTP 201)
```

**问题分析**：
- ❌ 记录类型需要手输，容易出错（CNAME vs CNAMe vs cname）
- ❌ 记录名的提示不够清晰，用户误传完整域名
- ❌ 报错信息没有说明"不能包含点号"这一关键信息
- ❌ 记录值的提示不够，用户不知道是否需要完整域名

---

## ✅ **改进后的流程**

```bash
=== 添加 DNS 记录 ===

请选择记录类型:
  1. A       - IPv4 地址
  2. AAAA    - IPv6 地址
  3. CNAME   - 规范名称          # 👈 清晰列出选项
  4. MX      - 邮件交换
  5. TXT     - 文本记录
  6. NS      - 名字服务器
  7. SRV     - 服务记录
  8. CAA     - 证书颁发机构

请输入选择 (1-8): 3               # ✅ 数字菜单，不易出错

请输入记录名 (@, www, api 等子域): blog
# ✅ 成功，不会再出错

请输入记录值 (完整域名，如 example.com): wordpress.com
# ✅ 提示清晰，告诉用户需要完整域名

请输入 TTL (默认 3600): 3600      # 或按 Enter

正在添加 CNAME 记录...            # 清晰的操作提示
✅ 操作成功 (HTTP 201)
```

**改进效果**：
- ✅ 菜单式选择，避免手输错误
- ✅ 类型说明清晰，用户一目了然
- ✅ 记录名提示明确："等子域"，不能有点号
- ✅ 记录值提示明确："完整域名，如 example.com"
- ✅ 操作反馈清晰："正在添加 CNAME 记录"

## 📞 **常见问题解答**

### Q: 为什么记录名不能有点号？
A: 因为 DNS 记录名是子域标识符（如 www、api），不是完整域名。
   完整域名应该放在"记录值"字段中。

### Q: 怎样添加 CNAME 记录到 blog.example.com？
A:
```bash
记录类型: CNAME（选 3）
记录名: blog           # 仅子域
记录值: example.com   # 完整域名或目标域名
```

### Q: MX 记录的优先级有什么用？
A: 决定邮件服务器的优先级。数字越小优先级越高（10 > 20）。
